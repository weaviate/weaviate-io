## Overview

This page will show you how to create individual objects in Weaviate. Note that whenever you want to create multiple objects, it's best to use [batch operations](./import.mdx).

:::info See also
- [How-to: (Batch) Import items](./import.mdx)
- [References: REST - /v1/objects](../api/rest/objects.md)
:::


## Requirements

To create an object, specify:
* The object to be added, and
* The target class.

### Object construction

You do not need to specify data for all defined object properties.

If you specify data for any new properties when creating the object, the [auto-schema feature](../config-refs/schema.md#autoschema) will add them to the schema during creation. If this isn't desired, you can [validate](#validating-before-creation) that the object does not contain any extra properties vs. the class definition.

## Preventing duplicates

To prevent duplicates, you can [efficiently check if an object exists](./read.mdx#check-object-existence) given its [id](../api/graphql/additional-properties.md#id). Note that Weaviate allows identical objects (in terms of property values), so the id is the only differentiator.


## Validating before creation

To make sure the creation of an object will succeed, and that it doesn't have any extra properties vs. the schema, you can [validate](../api/rest/objects.md#validate-a-data-object) an object against the class definition before creating it.
