## Overview

Each Weaviate object belongs to one (and only one) class. This page covers the essential topics in relation to managing Weaviate classes.

## Create a class

You can create a class:
- Manually, or
- Have Weaviate create it ("auto-schema") as needed.

We strongly suggest you create the class manually, as many of the settings in the class definition are immutable.

### Manual creation

To create a class, you need a class name at a minimum with the `class` parameter.

Typically, you would specify additional configurations as shown in the following sections.

### Sample configuration

One example configuration is shown below, which defines:
- The class name (`Article`)
- The vectorizer module (`text2vec-cohere`)
- The vector index distance metric (`cosine`)
- Which generative module to use (`generative-openai`)
- A set of properties (`title`, `chunk`, `chunk_no` and `url`), with data types set accordingly.
- Tokenization option for the `url` property.

```json
{
    "class": "Article",
    "vectorizer": "text2vec-cohere",
    "vectorIndexConfig": {
        "distance": "cosine",
    },
    "moduleConfig": {
        "generative-openai": {}
    },
    "properties": [
        {
            "name": "title",
            "dataType": ["text"]
        },
        {
            "name": "chunk",
            "dataType": ["text"]
        },
        {
            "name": "chunk_no",
            "dataType": ["int"]
        },
        {
            "name": "url",
            "dataType": ["text"],
            "tokenization": "field"
        },
    ],
}
```

You can specify as many of the available options in the class definition as you would like.

:::tip Where to find a list of available options
For a list of available options, we recommend starting at the [References: Configuration: Schema page](../config-refs/schema.md) page.
:::


### Auto-schema

If you create objects with a class that does not exist, Weaviate will create the class by inferring necessary information as required.

This may be fine in some circumstances, but we recommend you do not do this to avoid unwanted definitions.

:::info Read more about Auto-schema
See [this page](../config-refs/schema.md#auto-schema)
:::


## Example class configurations

We include a few sample class configurations below as starting points.

<details>
  <summary>For text objects and generative search</summary>

The below defines:
- The class name (`Article`)
- The vectorizer module (`text2vec-cohere`) and model (`multilingual-22-12`)
- The vector index distance metric (`cosine`)
- Which generative module to use (`generative-openai`) and temperature (`0.2`)
- A set of properties (`title`, `chunk`, `chunk_no` and `url`), with data types set accordingly.
- Tokenization option for the `url` property.
- Vectorization option (`skip` vectorization) for the `url` property.

```json
{
    "class": "Article",
    "vectorizer": "text2vec-cohere",
    "vectorIndexConfig": {
        "distance": "cosine",
    },
    "moduleConfig": {
        "text2vec-cohere": {
            "model": "multilingual-22-12",
        },
        "generative-openai": {
            "temperatureProperty": 0.2
        }
    },
    "properties": [
        {
            "name": "title",
            "dataType": ["text"]
        },
        {
            "name": "chunk",
            "dataType": ["text"]
        },
        {
            "name": "chunk_no",
            "dataType": ["int"]
        },
        {
            "name": "url",
            "dataType": ["text"],
            "tokenization": "field",
            "moduleConfig": {
                "text2vec-cohere": {
                    "skip": true
                },
            }
        },
    ],
}
```

</details>

<details>
  <summary>For image objects</summary>

```json
{
    "class": "FashionItem",
    "vectorizer": "img2vec-neural",
    "vectorIndexConfig": {
        "distance": "cosine",
    },
    "moduleConfig": {
        "img2vec-neural": {
            "imageFields": [
                "image"
            ]
        }
    },
    "properties": [
        {
            "name": "caption",
            "dataType": ["text"]
        },
        {
            "name": "image",
            "dataType": ["blob"]
        },
    ],
}
```

</details>
