Follow these steps to connect to a locally hosted Weaviate instance.

## No authentication enabled

By default Weaviate allows unauthenticated connections from `localhost`. You should always enable authentication for production instances.

Do not expose your Weaviate instance to the public internet if authentication is not enabled. If you do not enable authentication, anyone who has the URL can access your instance.

## Local connection URL

Docker instances default to `http://localhost:8080`. The gRPC port, `50051`, is also on `localhost`.

If your instance runs on Kubernetes, see the `host` and `port` values in your Helm chart's `values.yaml` file.
